# ⚠️ Important: Figma Make Environment Notes

## Files That Cannot Be Deleted

This project is developed in Figma Make, which includes system files that cannot be removed:

### 1. Supabase Infrastructure (Not Used in Production)
- `/supabase/` - Figma Make's Supabase templates
- `/utils/supabase/` - Supabase utilities (autogenerated)

**Note:** AccuraX uses a custom Node.js + Express + MySQL stack, NOT Supabase.

### 2. Figma Make System Files
- `/guidelines/Guidelines.md` - Template for AI guidelines

### 3. Orphaned Dockerfile Folders
- `/Dockerfile/` - Not used (separate frontend/backend deployments)
- `/backend/Dockerfile/` - Placeholder (real Dockerfile on GitHub)

## ✅ Deployment Protection

All above files are blocked by `.gitignore` and will NOT be deployed:

```gitignore
supabase/
utils/supabase/
guidelines/
Dockerfile/
backend/Dockerfile/
```

## 🚀 Actual Deployment Architecture

### Frontend: Hostinger
- URL: https://accurax.in
- Stack: React + TypeScript + Vite + Tailwind
- Deploy: Auto from GitHub main branch
- Source: `/src/` folder only

### Backend: Coolify  
- URL: https://api.accurax.in
- Stack: Node.js + Express + MySQL + Nodemailer + Razorpay
- Deploy: Docker container on Coolify
- Source: `/backend/` folder
- Dockerfile: Created manually on GitHub (not in Figma Make)

## 📝 Development Workflow

1. **Edit code in Figma Make** ✏️
2. **Push to GitHub** (`.gitignore` filters out system files) 🚀
3. **Auto-deploy to Hostinger** (frontend) 🌐
4. **Manual deploy to Coolify** (backend with Dockerfile) ⚙️

---

**Bottom Line:** System files exist in Figma Make but are invisible to production. Only clean, production-ready code reaches GitHub and deployment servers.
